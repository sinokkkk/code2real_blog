本次期末 Hackathon 项目基于 `F:\code_to_real\end\test1` 工程完成，主控为 ESP32-S2 Mini。项目目标是做一个小型智能灯光交互系统：网页和语音都可以控制双舵机动作，光敏模块自动控制 LED，OLED 根据环境状态显示表情或时间。

整体思路不是单独做一个网页控制舵机，也不是只做一个光敏灯，而是把网页控制、语音识别、光照检测、OLED 显示和双舵机动作整合到同一块 ESP32-S2 Mini 上。这样更接近一个完整的小作品，也更容易暴露嵌入式项目中常见的供电、接线、串口和交互问题。

## step1 项目选题与目标确定

期末项目最初的方向是“智能开关灯”。如果只用按钮或网页控制 LED，功能过于简单；如果完全依赖语音模块直接控制舵机，又很难统一调试。因此最终把 ESP32 放在系统中心，让它同时接收网页指令和 ASRPRO 语音识别结果，再统一控制舵机动作。

项目目标分为四部分：

- 网页端可以控制开灯、关灯、开左灯、开右灯等动作。
- ASRPRO 语音模块可以通过串口发送识别 ID。
- 光敏模块根据环境明暗自动控制 LED。
- OLED 显示表情或时钟，作为系统状态反馈。

这个设计的好处是每个模块都有明确职责：网页负责远程控制，语音模块负责自然交互，光敏模块负责环境感知，OLED 负责可视化反馈，双舵机负责执行动作。

## step2 硬件方案设计

硬件使用 ESP32-S2 Mini 作为主控，外接两个 SG90 舵机、一个 LED、四脚光敏模块、ASRPRO 语音识别模块、SSD1315/SSD1306 兼容 OLED，以及一个显示切换按钮。

主要接线如下：

| 模块 | ESP32-S2 Mini 引脚 |
|---|---:|
| 左舵机信号线 | GPIO7 |
| 右舵机信号线 | GPIO8 |
| LED | GPIO9 |
| 光敏模块 AO | GPIO1 |
| 光敏模块 DO | GPIO10 |
| 显示切换按钮 | GPIO12 |
| ASRPRO TX | GPIO16 |
| ASRPRO RX | GPIO17 |
| OLED SDA | GPIO33 |
| OLED SCL | GPIO35 |

舵机供电是硬件部分最需要注意的地方。两个 SG90 同时动作时电流波动比较明显，如果直接从 ESP32 板子取电，容易造成复位或舵机抖动。

## step3 第一版：先做 ESP32-S2 Mini 基础原型

开发不是一开始就完成最终版本，而是先做一个能跑的 ESP32-S2 Mini 原型。第一版目标比较直接：ESP32 连接 WiFi，网页控制一个 SG90 舵机，光敏电阻控制 LED，OLED 显示“高兴”和“困了”的表情。

这个阶段先选定了几个基础库：

```cpp
#include <ESP32Servo.h>
#include <U8g2lib.h>
#include <WebServer.h>
#include <WiFi.h>
#include <Wire.h>
```

最初的思路是先把“网页控制舵机”和“光敏 LED”跑通，再慢慢加 OLED 表情。这样做的原因很简单：如果一上来就把网页、语音、屏幕、双舵机全部塞进去，出问题时很难判断是哪一层坏了。

## step4 第二版：发现需要两个舵机，改成左右控制

第一版只考虑了一个 SG90，但宿舍里有两片，需要两个舵机，分别控制左边和右边。因此第二步把单舵机结构改成双舵机结构。

代码里从一个 `Servo` 扩展成两个对象：

```cpp
Servo leftServo;
Servo rightServo;
```

接线也随之确定：

```text
左舵机信号线 -> GPIO7
右舵机信号线 -> GPIO8
```

这一阶段网页还保留调试性质，使用左右两个角度控制入口，方便观察两个舵机是否能独立转动。调试时最容易出现的问题是左右方向反了，解决方式不是马上重接线，而是先调整代码顶部的角度常量。

解决后的表现是：左舵机和右舵机可以分别响应，串口输出也能看到两个舵机的角度状态。到这一步，项目从“单执行机构”变成了“双执行机构”。

## step5 重新检查光敏电阻：从裸 5516 到四脚模块

最开始光敏部分按裸 5516 分压电阻理解。后来检查后发现，实际使用的是四脚光敏模块，不是单独一颗光敏电阻。四脚模块通常带 LM393 比较器，有 `VCC/GND/AO/DO` 四个引脚。

这就解释了前期 LED 经常闪动、光敏模块看起来“没作用”的问题：程序按模拟量阈值判断，但实际接线和模块输出可能更适合用 DO 数字口。

后续接线改成：

```text
光敏模块 VCC -> 3.3V
光敏模块 GND -> GND
光敏模块 AO  -> GPIO1
光敏模块 DO  -> GPIO10
LED          -> GPIO9
```

程序默认优先使用 DO：

```cpp
const bool USE_LIGHT_DIGITAL_OUTPUT = true;
const bool LIGHT_DIGITAL_ACTIVE_LOW = false;
```

同时加入连续读取过滤：

```cpp
const int LIGHT_STABLE_READS = 4;
```

解决后的表现是：遮住光敏模块时，LED 能稳定点亮；移开光照后，LED 能稳定熄灭；在临界光线下不会一直闪。

## step6 加入网页控制说明：ESP32 网页服务器

光敏部分稳定后，继续确认网页控制逻辑。ESP32 连接 WiFi 后，在本地开启 `WebServer server(80)`，手机或电脑只要和 ESP32 在同一个 WiFi 下，就能访问串口打印出的地址。

串口正常输出类似：

```text
 http://192.168.79.247
```

## step7 讨论 ASRPRO：先确认它不能和 ESP32 抢舵机

后续加入 ASRPRO 语音模块时，ASRPRO 和 ESP32 如果同时给同一个舵机发 PWM，舵机就会抖动。因此最终确定原则：

```text
ASRPRO 只负责识别语音
ESP32 负责唯一控制舵机
```

也就是说，ASRPRO 不接舵机信号线，只通过串口把识别结果发给 ESP32。舵机仍然接在 ESP32 上，这样网页控制和语音控制可以共存。

## step8 ESP32 解析语音 ID：网页和语音共用动作

ESP32 端新增 `HardwareSerial asrSerial(1)`，专门接收 ASRPRO 串口数据。程序使用 `parseAsrId()` 从字符串中提取数字，因此不仅能识别 `3`，也能识别 `ID=3` 这类格式。

语音动作和网页动作最后都进入同一个函数：

```cpp
bool runServoAction(int id) {
  switch (id) {
    case 1:
      tapBothServos(LEFT_SERVO_ON_ANGLE, RIGHT_SERVO_ON_ANGLE);
      return true;
    case 2:
      tapBothServos(LEFT_SERVO_OFF_ANGLE, RIGHT_SERVO_OFF_ANGLE);
      return true;
    case 3:
      tapLeftServo(LEFT_SERVO_ON_ANGLE);
      return true;
    case 4:
      tapRightServo(RIGHT_SERVO_ON_ANGLE);
      return true;
    case 5:
      tapRightServo(RIGHT_SERVO_OFF_ANGLE);
      return true;
    case 6:
      tapLeftServo(LEFT_SERVO_OFF_ANGLE);
      return true;
    default:
      return false;
  }
}
```

这里还处理了一个细节：ASRPRO 有时输出没有换行。如果程序只等 `\n`，命令可能不会执行。因此加入 `ASR_LINE_TIMEOUT_MS`，收到字符后一小段时间没有新字符，也会尝试解析。

解决后的表现是：网页按钮和语音命令触发的是同一套舵机动作，后续维护只需要改一个动作函数。

## step9 OLED 显示调整：复杂颜文字显示不全

后来尝试把开心表情换成 `ヾ(≧▽≦*)o`，但实际显示不全。问题主要有两个：一是 128 像素宽度不够，二是 U8g2 字体对复杂字符支持有限。

解决方式是换成更适合 OLED 的 ASCII 表情：

```text
(^///^)
```

最终 OLED 逻辑确定为：

```text
亮 / LED 灭：(^///^)
暗 / LED 亮：-_- zZ
```

这一版牺牲了一点花哨程度，但稳定显示更重要。

## step10 加入按钮切换时钟模块

在表情显示稳定后，又加入了一个按钮切换的时钟模块。按钮接在 GPIO12，采用 `INPUT_PULLUP`：

```text
GPIO12 -> 按钮 -> GND
```

显示模式变成两种：

```text
DISPLAY_FACE
DISPLAY_CLOCK
```

默认是光敏表情模式，按一下按钮切到时钟模式，再按一下切回表情模式。时钟来自 WiFi/NTP。

## step11 当前完成效果

截至期末 Hackathon 阶段，项目已经完成以下功能：

- ESP32-S2 Mini 成功连接 WiFi，并开启网页控制面板。
- 网页按钮可以控制两个舵机完成开灯、关灯等动作。
- ASRPRO 可以通过串口 ID 触发同一套舵机动作。
- 光敏模块可以根据环境明暗自动控制 LED。
- OLED 可以显示 `(^///^)` 和 `-_- zZ`，也可以通过按钮切换到时钟页面。
- 串口能持续输出光敏值、LED 状态、舵机角度和当前状态。

## step14 项目总结

这个期末项目不是一次写完的，而是按问题一步步推进出来的。先做 ESP32 基础原型，再改双舵机；中间改过 ESP8266，又回到真实板卡；光敏部分从裸 5516 思路改成四脚模块逻辑；ASRPRO 从“直接控制舵机”改成“只发识别 ID”；OLED 从圆脸改成稳定颜文字；网页最后也从调试滑条收紧成动作按钮。

最终方案采用 ESP32 作为统一控制中心。网页和语音只负责发送动作 ID，舵机动作统一由 `runServoAction()` 执行；光敏模块独立决定 LED 状态；OLED 负责表情和时钟反馈。这样系统结构比较清楚，也便于后续维护。

从 Hackathon 结果看，项目已经达到期末展示要求。它不是单一传感器实验，而是一个经过多轮修正后形成的小型嵌入式交互系统。
